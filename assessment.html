Sobre Design Responsivo

  1. Quando é recomendado a criação de um site separado para mobile? Seria este o casa deste site da Câmara dos Deputados?

    Resposta: A criação de um site separado para mobile é recomendada quando a performance é um requisito fundamental, já que normalmente sites para mobile são bem mais rapidos devido a sua implementação ser mais específica (pouca responsividade) o que torna seus arquivos menores. Pesquisa mostram que sites mobile chegam a ser carregados 60% mais rapido que sites responsivos [Ref. 1]. No site que criamos não há essa necessidade devido ao site ser pequeno e o ganho de performance seria ínfimo comparado ao esforço de desenvolvimento.

  2. No caso de um CSS específico para impressão (print), como ele deve ser inserido?

    Resposta: Pode-se criar uma folha de estilos separada e incorporá-la ao HTML adicionando a propriedade media="print" na incorporação. Outra opção é, utilizando folhas de estilos já existentes, adicionar uma media query para o media type print (Ex: @media print {...}).

  1. O elemento audio permite como elemento-filho o elemento source. Quais são os atributos que o source permite e qual conteúdo eles suportam? Quais os outros atributos do elemento audio e quanto devem ser utilizados?

    Resposta: Atributos do elemento source [Ref. 2].

    Atributo Valor Descrição
    media media query Aceita qualquer media query que poderia ser definido no CSS
    size Especifica o tamanho da imagem para diferentes layouts de pagina
    src URL Especifica a URL do arquivo de mídia. Esse atributo é obrigatório quando a tag source é utilizada nas tags video e audio.
    srcset URL Especifica a URL da imagem que será utilizada em diferentes situações. Esse atributo é obrigatório quando a tag source é utilizada nas tags video e audio.
    type MIME-type Especifica o MIME-type do recurso

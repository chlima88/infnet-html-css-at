Sobre Design Responsivo

  1. Quando é recomendado a criação de um site separado para mobile? Seria este o casa deste site da Câmara dos Deputados?

    Resposta: A criação de um site separado para mobile é recomendada quando a performance é um requisito fundamental, já que normalmente sites para mobile são bem mais rapidos devido a sua implementação ser mais específica (pouca responsividade) o que torna seus arquivos menores. Pesquisa mostram que sites mobile chegam a ser carregados 60% mais rapido que sites responsivos [Ref. 1]. No site que criamos não há essa necessidade devido ao site ser pequeno e o ganho de performance seria ínfimo comparado ao esforço de desenvolvimento.

  2. No caso de um CSS específico para impressão (print), como ele deve ser inserido?

    Resposta: Pode-se criar uma folha de estilos separada e incorporá-la ao HTML adicionando a propriedade media="print" na incorporação. Outra opção é, utilizando folhas de estilos já existentes, adicionar uma media query para o media type print. Ex: @media print {...}

Sobre multimídia Web

  1. O elemento audio permite como elemento-filho o elemento source. Quais são os atributos que o source permite e qual conteúdo eles suportam? Quais os outros atributos do elemento audio e quanto devem ser utilizados?

    Resposta: Atributos do elemento source [Ref. 2].

    Atributo Valor Descrição
    media media query Aceita qualquer media query que poderia ser definido no CSS
    size Especifica o tamanho da imagem para diferentes layouts de pagina
    src URL Especifica a URL do arquivo de mídia. Esse atributo é obrigatório quando a tag source é utilizada nas tags video e audio.
    srcset URL Especifica a URL da imagem que será utilizada em diferentes situações. Esse atributo é obrigatório quando a tag source é utilizada nas tags video e audio.
    type MIME-type Especifica o MIME-type do recurso

    Atributos do elemento audio [Ref. 3]

    Atributo Descrição
    autoplay Especifica que o audio vai começar a tocar assim que estiver pronto
    controls Especifica que os controles de audio devem ser mostrados (play, pause, etc)
    loop Especifica que o audio vai começar novamente, cada vez que terminar
    muted Especifica que a saída de audio deve ser silenciada
    preload Especifica se e como o audio deve ser carregado quando a pagina carregar
    src Especifica a URL do arquivo de audio

  2. O elemento vídeo permite como elemento-filho o elemento source. Quais são os atributos que o source permite e qual conteúdo eles suportam? Quais os outros atributos do elemento vídeo e quando devem ser utilizados?

    Resposta: Atributos do elemento source [Ref. 4]

    Atributo Valor Descrição
    media media query Aceita qualquer media query que poderia ser definido no CSS
    size Especifica o tamanho da imagem para diferentes layouts de pagina
    src URL Especifica a URL do arquivo de mídia. Esse atributo é obrigatório quando a tag source é utilizada nas tags video e audio.
    srcset URL Especifica a URL da imagem que será utilizada em diferentes situações. Esse atributo é obrigatório quando a tag source é utilizada nas tags video e audio.
    type MIME-type Especifica o MIME-type do recurso

    Atributos do elemento vídeo [Ref. 5]

    Atributo Descrição
    autoplay Especifica que o video vai começar assim que estiver disponível
    controls Especifica que os controles de vídeo devem ser mostrados (plat, pause, etc)
    height Define a altura do player
    loop Especifica que o vídeo vai começar novamente, toda vez que terminar.
    muted Especifica que a saída de audio do video deve ser silenciada
    poster Especifica uma imagem a ser exibida enquando o vídeo está sendo baixado ou até que o usuario clique no botão play
    preload Especifica se e como o video deve ser carregado quando a pagina carregar
    src Especifica a URL do arquivo de vídeo
    width Define a largura do player

  3. No caso do canvas, HTML5 nunca, em momento algum, fez qualquer animação. Isso porque o HTML5 é uma simples linguagem de marcação. Dessa forma, o que é necessário para criar animações com HTML5? O que são APIs do HTML5?

    Resposta: O canvas do HTML5 é apenas uma API com a qual podemos interagir através de códigos javascript [Ref. 6]. São instruções disponíveis nos navegadores que facilitam o acesso a funcionalidades complexas através da abstração de código.[Ref. 4]

  4. Como o CSS3 pode ser aplicado na aparência da multimídia HTML5. Demonstre com um exemplo.

    Resposta: Podemos utilizar o CSS3 para ajustar as dimensões do player de video. Ex: video {width: 100%; height: auto; }

Animações

As animações utilizadas foram através da propriedade "transition" (que é suportada por todos os browsers desde 2010) em conjuto com propriedades como filter (para o efeito de brilho nos botões), scale para redimencionar as imagens dos deputados e partidos. Outro recurso que utilizei as variáveis CSS que são totalmente suportadas apenas após 2016. Também utilizei as propriedades margin-inline, margin-block, padding-inline e padding-block que são totalmente suportadas apenas a partir de 2020.

Utilizei por ultimo uma técnica que gera um efeito parallax, que dá a sensação de tridimensionalidade entre os elementos da página. No caso desse projeto, esse efeito foi utilizado apenas na pagina "assessment.html", aonde pode-se observar o background rolando em uma velocidade diferente dos demais elementos. No entanto não consegui fazer esse efeito ficar totalmente responsivo.
